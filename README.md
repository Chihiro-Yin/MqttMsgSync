# MQTT 高性能消息推送系统（Kotlin 实现）
一款**低延迟、高并发**的跨平台消息推送与设备同步解决方案，基于标准 MQTT 协议构建，采用 Kotlin 原生开发，JSON 结构化消息传输，支持移动端、嵌入式设备、物联网终端无缝集群接入，实现多设备实时消息同步、指令下发、状态互通，是物联网、多端协同、消息中继的最佳实践。

## 核心优势
- **极致性能**：高并发连接支撑、毫秒级消息传输、超低资源占用，满足大规模设备同时在线与高频通信需求
- **通用接入**：完美兼容 Android 端 + 各类嵌入式设备（STM32、ESP32、Arduino、Linux 物联网终端等），跨硬件、跨系统统一通信
- **标准化消息**：采用 **JSON 格式** 传输所有业务数据，结构清晰、易解析、易扩展，支持自定义消息类型与业务字段
- **多设备实时同步**：验证码自动同步、设备状态实时共享、跨端指令中继、全局消息广播，一设备发送，多终端即时接收
- **稳定可靠**：MQTT 长连接保活、断线自动重连、消息 QoS 质量保证，弱网环境依然稳定运行
- **纯 Kotlin 实现**：代码简洁优雅、易于二次开发、维护成本低，Android 平台深度适配

## 功能亮点
- 多设备验证码/通知消息全局同步
- 嵌入式设备与移动端双向指令通信
- 高并发集群接入，低延迟实时推送
- JSON 结构化消息，支持自定义业务扩展
- 自动重连、心跳保活、异常容错机制
- 可视化权限管理与连接状态监控

## 快速部署 & 使用教程
### 1. 配置 MQTT 服务器地址
在项目核心配置文件中绑定你的私有/公有 MQTT 服务器：app/src/main/……/model/Model.kt
打开文件后配置服务器地址（示例：`tcp://mqtt.yourdomain.com:1883`），支持 TCP/SSL 加密连接。

### 2. 项目编译构建
使用 Android Studio 编译项目，自动拉取依赖库，完成后生成可安装 APK。
编译前确保项目环境配置正常，无依赖缺失。

### 3. 应用权限授予
安装并启动应用 → 进入**设置页面**，**必须授予以下权限**以保证 MQTT 消息推送正常运行：
- 网络访问权限（MQTT 连接必需）
- 后台运行权限（保活长连接）
- 通知权限（消息提醒）

### 4. 开始使用
权限配置完成后，系统自动连接 MQTT 服务器，即可实现：
- 多端设备消息实时同步
- 嵌入式设备 JSON 指令收发
- 高并发低延迟消息推送

## 技术架构
- 开发语言：Kotlin（Android 原生）
- 通信协议：MQTT
- 消息格式：**JSON 结构化传输**
- 支持设备：Android 移动端、嵌入式 IoT 设备、Linux 终端
- 核心特性：长连接、自动重连、QoS 消息保证、高并发、低延迟

## 典型应用场景
- 多设备验证码自动同步（手机/平板/车机同步接收短信/验证码）
- 物联网嵌入式设备远程控制（ESP32/STM32 与 APP 双向 JSON 通信）
- 多端协同消息系统（跨设备通知、状态同步）
- 工业物联网数据采集与指令下发
- 高并发低延迟即时通信中继服务
